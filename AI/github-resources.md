DeepLearning AI 联合打造，Coursera 全新课程《RAG（检索增强生成）》来啦！ 可深入学习如何构建高性能、可投入生产的 RAG 系统。

RAG 是当今许多基于 LLM 的应用（如客户支持、公司内部问答系统，甚至许多使用网页搜索来回答问题的主流聊天机器人）的关键组件。本课程将深入教授你如何让 RAG 良好运作。

LLM 可能会生成泛泛而谈或过时的回答，尤其是在被问到其训练数据中未涵盖的专业性问题时。RAG 是解决这一问题最广泛使用的技术。它从新的数据源（如内部文档或最新新闻）中引入数据，为 LLM 提供与私有、最新或专业信息相关的上下文。这使其能够生成更有依据、更准确的回答。

在本课程中，你将学习如何设计和实现 RAG 系统的每一个部分，从检索器到向量数据库再到生成模块和评估机制。你将学习 RAG 背后的基本原理，以及如何在组件级别和系统整体层面对其进行优化。

随着 AI 的发展，RAG 也在演进。新模型可以处理更长的上下文窗口，更有效地进行推理，并能作为复杂代理工作流的一部分。一个令人兴奋的增长领域是 Agentic RAG，其中 AI 代理在运行时（而不是在开发时被硬编码）自主决定检索哪些数据、何时/如何深入检索。即便在这种演进下，在运行时获取高质量数据仍然是关键，这就是 RAG 在众多应用中如此重要的原因。

你将通过实践体验学习以下内容：
- 构建具有检索和提示增强功能的 RAG 系统

- 比较检索方法，如 BM25、语义搜索和互惠排序融合（Reciprocal Rank Fusion）

- 使用 Weaviate 向量数据库和新闻数据集对文档进行切分、索引和检索

- 开发一个聊天机器人，使用 Together AI 托管的开源 LLM，针对虚拟商店回答产品和常见问题

- 使用评估机制推动可靠性改进，并整合多模态数据

访问：www.coursera.org/learn/retrieval-augmented-generation-rag

转：LinkedIn 上有人分享了如何构建 99% 代码都是 AI 来写的系统。系统性思维很棒，翻译了一下：

到现在我已经用 Claude Code 和 Copilot 来写代码差不多一个月了，基本就是我自己几乎不写——99.9% 的代码都是 AI agent 写的。下面是我能靠 AI 干成的事情，以及大概的花费：

Claude Code：大约 12.5 亿 Tokens，花了 ~640 美元  
Copilot：1375 次高级请求，企业版（39 美元/月）  
Claude Chat：几百次对话，同样是企业版（应该和 Copilot 一样）

代码库的变化：  
 * 新增 197K 行代码  
 * 删除 208K 行代码  
 * 主分支上有 355 次提交  
 * 合并了 180 个 PR  
 * 解决了 98 个 GitHub issue  

我是怎么做到的？

1) 所有代码都必须过一套非常严格的 CI/CD 流程：静态检查、格式化、代码安全工具，还有完整的测试集  
2) 代码库已经模块化了，每个组件都通过清晰的接口来调用  
3) 接口和实现完全分开  
4) 每个组件都有自己的文档  
5) 每个组件的测试、文档、头文件、实现都放在一起  
6) 每个组件保持在 5k 行代码以下（包括文档）  
7) 每个 GitHub issue 都写得很详细，带代码引用和问题描述  
8) 搭建了一个 7 层的提示体系，记录各种规范：工具层（Copilot vs Claude）、语言层（C++ vs Python）、项目层（按仓库）、角色层（开发 vs 测试 vs 架构 vs 审查）、组件层（按目录）、任务层（对应 GitHub issue）、执行层（具体操作）  

这样，我只需要分配任务，15-30 分钟就能拿到第一版，然后我再迭代、审查、验证，最后一小时内就能提交——几乎不用我亲手敲代码。  

现在真正的瓶颈是我自己，因为我审查代码的速度跟不上。  

而且 Claude Code 现在已经包含在企业计划里了。是时候继续玩一玩，让我的“Agent 军团”来帮我缓解这个瓶颈。

Claude 的 Agent Skills 本质上就是：AI 模型不必把所有能力硬塞在绝对静态的“大脑”里，而是通过“可插拔技能模块”的方式，让模型在面对不同任务时“辅以专属插件／套路”。这种方式在软件工程里早就被验证为经典，比如插件系统、微服务、模块化设计。

可以关注 Antropic 这个开源的 Skills 项目：github.com/anthropics/skills
「Skills 是一组“指令、脚本、资源”的集合，Claude可以动态载入这些 Skills，从而在特定任务或领域里表现得更好、更定制化」
